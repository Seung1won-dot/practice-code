# CSS (Cascading Style Sheets)

## CSS란?

CSS는 캐스케이딩 스타일 시트(Cascading Style Sheets)의 약자로, 웹 페이지의 스타일과 레이아웃을 정의하는 스타일시트 언어입니다. HTML이 웹 페이지의 구조를 담당한다면, CSS는 그 구조에 디자인과 스타일을 적용합니다.

## CSS의 특징

### 1. 스타일 분리
- HTML에서 스타일을 분리
- 구조와 표현의 분리
- 유지보수성 향상

### 2. 캐스케이딩 (Cascading)
- 여러 스타일 규칙이 적용될 때 우선순위 결정
- 상위 요소의 스타일이 하위 요소에 상속
- 폭포수처럼 스타일이 흐르는 구조

### 3. 선택자 시스템
- 요소, 클래스, ID 선택자
- 복합 선택자와 의사 선택자
- 정밀한 스타일 타겟팅

### 4. 반응형 디자인
- 미디어 쿼리를 통한 반응형 레이아웃
- 다양한 화면 크기 지원
- 모바일 퍼스트 접근

### 5. 애니메이션과 전환
- CSS 애니메이션
- 트랜지션 효과
- JavaScript 없이도 동적 효과 가능

## CSS의 활용 분야

### 웹 페이지 스타일링
- 레이아웃 디자인
- 색상, 폰트, 간격 조정
- 반응형 웹 디자인

### 웹 애플리케이션 UI
- 컴포넌트 스타일링
- 테마 시스템
- 다크 모드 지원

### 프린트 스타일
- 인쇄용 스타일시트
- 페이지 브레이크 제어
- 인쇄 최적화

### 애니메이션
- CSS 애니메이션
- 호버 효과
- 로딩 애니메이션

## CSS 버전

### CSS1
- 1996년 첫 번째 표준
- 기본 스타일링 기능

### CSS2
- 1998년 표준화
- 포지셔닝, 미디어 타입 추가

### CSS2.1
- 2011년 표준화
- CSS2의 개선 버전

### CSS3
- 모듈화된 표준
- Flexbox, Grid, 애니메이션 등
- 현재 널리 사용되는 버전

### CSS4
- 계속 발전 중
- 새로운 선택자와 기능 추가

## 학습 경로

### 기초 단계
1. CSS 기본 문법
2. 선택자 (요소, 클래스, ID)
3. 색상, 폰트, 텍스트 스타일
4. 박스 모델 (margin, padding, border)
5. 배경과 이미지

### 중급 단계
1. 레이아웃 (float, position)
2. Flexbox
3. Grid
4. 반응형 디자인 (미디어 쿼리)
5. 변수 (CSS Custom Properties)

### 고급 단계
1. 애니메이션과 전환
2. 고급 선택자
3. CSS 아키텍처 (BEM, OOCSS)
4. 전처리기 (Sass, Less)
5. 성능 최적화

## CSS 개발 도구

### 전처리기
- **Sass/SCSS**: 가장 인기 있는 전처리기
- **Less**: JavaScript 기반
- **Stylus**: 간결한 문법

### 후처리기
- **PostCSS**: 플러그인 기반
- **Autoprefixer**: 벤더 프리픽스 자동 추가

### 프레임워크
- **Bootstrap**: 가장 널리 사용
- **Tailwind CSS**: 유틸리티 퍼스트
- **Bulma**: 현대적인 프레임워크

### 개발 도구
- **Chrome DevTools**: 브라우저 개발자 도구
- **CSS Validator**: W3C 검증 도구
- **Can I Use**: 브라우저 호환성 확인

## CSS의 핵심 개념

### 박스 모델
- content, padding, border, margin
- box-sizing 속성
- 요소 크기 계산

### 레이아웃
- Normal Flow
- Float
- Position (static, relative, absolute, fixed, sticky)
- Flexbox
- Grid

### 선택자 우선순위
- 중요도 (importance)
- 명시도 (specificity)
- 코드 순서 (source order)

### 캐스케이딩
- 스타일 상속
- 우선순위 규칙
- !important 키워드

## CSS 작성 방법

### 인라인 스타일
- HTML 태그에 직접 style 속성 사용
- 우선순위가 가장 높음
- 유지보수가 어려움

### 내부 스타일시트
- `<style>` 태그 사용
- HTML 문서 내에 포함
- 작은 프로젝트에 적합

### 외부 스타일시트
- 별도 .css 파일로 분리
- `<link>` 태그로 연결
- 가장 권장되는 방법

## CSS의 장점

1. **분리**: 구조와 스타일의 분리
2. **재사용성**: 한 스타일을 여러 요소에 적용
3. **유지보수**: 중앙 집중식 스타일 관리
4. **성능**: 캐싱으로 로딩 속도 향상
5. **일관성**: 전체 사이트의 일관된 디자인

## CSS의 단점

1. **복잡성**: 복잡한 레이아웃 구현이 어려움
2. **브라우저 차이**: 브라우저마다 렌더링 차이
3. **디버깅**: 스타일 오류 찾기가 어려울 수 있음
4. **명시도**: 선택자 우선순위로 인한 혼란
5. **제한사항**: 일부 디자인 구현의 한계

## CSS와 다른 기술의 관계

### HTML
- HTML: 구조
- CSS: 스타일
- 함께 사용하여 완성된 웹 페이지

### JavaScript
- CSS: 정적 스타일
- JavaScript: 동적 스타일 변경
- 클래스 토글, 스타일 조작

### Sass/SCSS
- CSS 전처리기
- 변수, 중첩, 믹스인 제공
- CSS로 컴파일됨

## 현대 CSS 기능

### Flexbox
- 1차원 레이아웃
- 유연한 정렬과 분배
- 반응형 디자인에 적합

### Grid
- 2차원 레이아웃
- 복잡한 그리드 시스템
- 정밀한 레이아웃 제어

### CSS 변수
- Custom Properties
- 동적 테마 변경
- 재사용 가능한 값

### 컨테이너 쿼리
- 요소 기반 반응형 디자인
- 컨테이너 크기에 따른 스타일
- 최신 CSS 기능

## 결론

CSS는 웹 디자인의 핵심 언어입니다. HTML과 함께 사용하여 아름답고 기능적인 웹 페이지를 만들 수 있습니다. 캐스케이딩과 선택자 시스템을 이해하고, Flexbox와 Grid 같은 현대적 기능을 활용하면 효율적이고 반응형인 웹 디자인을 구현할 수 있습니다. 지속적으로 발전하는 언어로, 최신 기능들을 학습하여 현대적인 웹 개발을 할 수 있습니다.
